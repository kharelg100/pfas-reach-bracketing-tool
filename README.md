# PFAS Reach Bracketing Tool — Browser Edition

Version: **1.1.1 browser + Python bundle**  
Archived software DOI: **10.5281/zenodo.20464907**  
Concept DOI: **10.5281/zenodo.18937739**

This browser tool implements the manuscript-aligned PFAS reach-bracketing workflow. It is distributed with a companion Python edition for Spyder/Anaconda, Jupyter, and Google Colab workflows.

Developed by **Dr. Gehendra Kharel, STREAM Lab (Sustainable Tools for Risk Evaluation and Climate-Water Modeling), Texas Christian University**. It converts long-format PFAS results, optional site metadata, and a machine-readable reach experiment table into:

- censoring-aware site totals for `ΣPFAS40`,
- 95% multiple-imputation intervals,
- RL/2 and detect-only sensitivity summaries,
- reach-scale `ΔΣPFAS40` contrasts,
- concentration-space `f_trib` diagnostics where applicable,
- convergence diagnostics, and
- downloadable CSV validation outputs.

All computation is performed locally in the user's browser. No uploaded data are transmitted.

## Python edition

The bundled Python script can be downloaded from the browser interface or run directly from the release folder:

```bash
python code/PFAS_Reach_Bracketing_Tool.py
```

When run from this browser package, the script automatically uses `data/` as the input folder if `inputs/` is not present. Outputs are written to `outputs/`.

## Input schema

### PFAS results CSV
Required columns:

```text
site_id, campaign, analyte_std, result_value, detect_flag, rl
```

Optional columns such as `qualifier`, `campaign_date`, `client_sample_id`, `sample_id`, `result_text`, and `unit` are retained where possible.

### Sites metadata CSV
Recognized columns:

```text
Site ID or site_id, Lat, Lon, Reach_group, Bracket_role, USGS_streamflow_gage
```

If no sites metadata file is supplied, the tool auto-generates a site list from the PFAS results.

### Reach experiment CSV
Recommended manuscript schema:

```text
experiment_id, reach_group, experiment_type, downstream_site, upstream_sites,
tributary_sites, baseline_sites, all_sites
```

Use pipe-delimited entries for multi-site roles, e.g. `TW82|TW72`.

## Methods implemented

- Default multiple imputation replicates: **5,000**.
- Random seed default: **20260227**.
- For analytes with ≥2 detections, non-detects are imputed from a fitted lognormal distribution truncated at `[0, RL]`.
- For analytes with <2 detections, non-detects are imputed from `Uniform(0, RL)`.
- `f_trib` is computed as:

```text
f_trib = (C_dn - C_up) / (C_trib - C_up)
```

`f_trib` is a concentration-space screening diagnostic only. It is not a discharge fraction, source-apportionment fraction, or load fraction. The tool reports `n/a` when the downstream concentration is outside the upstream–tributary end-member range or for experiment types where `f_trib` is not applicable.

## Validation

The bundled demo data reproduce the manuscript’s key results within Monte Carlo tolerance. A JavaScript PRNG is used in the browser edition, so medians and intervals may differ slightly from the Python edition, but screening conclusions and rankings are unchanged.

Validation highlights with the bundled data (`n = 5,000` MI replicates):

- 20 site-campaign samples
- 15 unique sites
- 40 target analytes
- 40 rows excluded because Campaign 1 TW04 is outside the GK2v3 network
- Spearman rho, MI vs RL/2: ~1.0000
- Campaign 2 TW91 `ΣPFAS40`: ~15,667 ng L-1
- Campaign 2 WF1 Farmers Branch `ΔΣPFAS40`: ~728 ng L-1
- Campaign 2 WF1 `f_trib`: ~0.047
